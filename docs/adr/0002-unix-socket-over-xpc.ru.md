# ADR 0002 — Unix domain socket для IPC вместо XPC

* **Статус:** Принято (Phase 1)
* **Дата:** 2026-05-06

## Контекст

Демону нужен интерфейс для клиентов — как внутрипроцессных, так и
внешних (будущий MenuBar UI, CLI-инструменты, скрипты, сторонние
интеграции).

Варианты:

1. **NSXPC / `xpc_main`.** Рекомендованный Apple путь для первичных macOS-демонов.
   Требует зарегистрированного в launchd Mach-имени сервиса, подписанного bundle,
   Info.plist, а на практике — ещё настройки sandbox + entitlements, чтобы
   инструментарий Apple был доволен.
2. **AF_UNIX SOCK_STREAM** по известному пути внутри
   `~/Library/Application Support/Froggy/`. Контроль доступа через биты
   прав доступа файловой системы.
3. **TCP/HTTP на localhost.** Проще всего подключиться с любого языка, но
   открывает порт, требует думать о firewall и не передаёт peer credentials.

## Решение

Unix domain socket. Путь настраивается через `FroggyConfig.ipcSocketPath`,
по умолчанию `~/Library/Application Support/Froggy/froggy.sock` с правами
`0600`. Протокол — один JSON-объект на строку в каждом направлении
(`IPCRequest`, `IPCResponse`).

## Последствия

* **Плюс:** Не нужны bundle, регистрация Mach-сервиса, code-signing для
  *разработки*. `swift run FroggyDaemon` с последующим
  `nc -U …/froggy.sock` работает сразу.
* **Плюс:** Тривиально написать клиент с любого языка (Python, Node, Bash
  через `socat`).
* **Плюс:** Filesystem ACL достаточно, чтобы закрыть доступ другим
  пользователям — права 0600 + сокет живёт в `~/Library` текущего пользователя.
* **Минус:** Нет ARC / разделения Sendable-типов через границу; протокол
  строково-типизирован через JSON. Смягчаем это единой парой
  `IPCRequest`/`IPCResponse` с Codable и тонким actor-ом `IPCClient` в
  `VortexCore`, который другие Swift-потребители могут импортировать.
* **Минус:** Нет проверки peer credentials помимо прав файловой системы.
  Если когда-нибудь потребуется открыть Froggy в многопользовательской
  системе — понадобятся проверки в стиле `SO_PEERCRED`.
* **Минус:** Нет потоковых ответов (пока). Долгая генерация приходит
  единым блоком ответа. Кандидат для Phase 4: chunked-ответы с маркером
  потокового протокола.

## Рассмотренные альтернативы

* **XPC через `NSXPCConnection`.** Можно пересмотреть, когда выйдет
  полноценный подписанный установщик; демон и UI получат небольшую XPC-обёртку
  поверх того же протокола `IPCRequestHandler`.
* **gRPC.** Избыточно для демона личного пользования; добавляет protobuf +
  кодогенерацию.
