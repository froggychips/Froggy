# ADR 0003 — Codable JSON для хранения конфигурации вместо TOML/YAML

* **Статус:** Принято (Phase 1)
* **Дата:** 2026-05-06

## Контекст

Froggy нужны постоянные настройки (путь к модели, лимит GPU-памяти,
интервал OCR, allowlist для заморозки, путь к IPC-сокету, порог разницы
кадров, размер контекстного окна).

Распространённые варианты:

1. **TOML** — удобно редактировать вручную; требует стороннего Swift-парсера
   (например, `dduan/TOMLDecoder`).
2. **YAML** — те же преимущества при ручном редактировании, плюс ловушки
   с отступами и более тяжёлый парсер.
3. **JSON** с `Codable` — многословен при ручном редактировании, нулевые
   зависимости, точно roundtrip-ит ту же структуру, которую использует
   остальная часть демона.

## Решение

`FroggyConfig: Codable, Sendable, Equatable`, сохраняемый как JSON в
`~/Library/Application Support/Froggy/config.json` с правами `0600`.
Кастомный `init(from:)` откатывается к значениям по умолчанию для каждого
поля, поэтому конфиг, записанный старой версией, корректно загружается
при добавлении новых полей.

## Последствия

* **Плюс:** Нет новых зависимостей SPM; меньше поверхности для проверки
  безопасности.
* **Плюс:** Та же структура является единственным источником истины для
  тестов, значений по умолчанию и дискового формата — добавлять поле
  нужно в одном месте.
* **Плюс:** Прямая совместимость: отсутствующие ключи → значения по умолчанию
  через `decodeIfPresent`.
* **Минус:** Ручное редактирование JSON слегка болезненно (нет комментариев,
  строгие запятые). Смягчаем это через `FroggyConfig.save()` и приложение
  MenuBar.
* **Минус:** Нет валидации схемы помимо проверок типов Codable. Приемлемо,
  поскольку конфиг является пользовательским и мы контролируем его
  производителя.

## Рассмотренные альтернативы

* **TOML.** По-настоящему удобнее для ручного редактирования пользователем;
  стоит пересмотреть, если появится CLI-ориентированный процесс установки
  без MenuBar UI.
* **plist.** Нативен для macOS, но редактируется вручную даже хуже, чем JSON,
  и вводит XML или бинарную обработку, с которой Codable в JSON отлично
  справляется сам.
