# Политика безопасности

## Сообщение об уязвимостях

По вопросам безопасности обращайтесь напрямую в Telegram
[@froggychips](https://t.me/froggychips), а не открывайте публичный issue
на GitHub. Достаточно короткого сообщения с шагами воспроизведения; ответ
следует ожидать в течение нескольких дней. Программы вознаграждений нет —
это личный проект, — однако за содержательные сообщения предлагается упоминание
в release notes.

По несекьюрити-багам используйте GitHub Issues.

## Модель угроз

Froggy разработан исходя из предположения, что **локальный пользователь
не является злоумышленником**. Это единственная поддерживаемая конфигурация.

В скоупе:

- Устойчивость к случайному некорректному использованию (неправильно
  сформированные IPC-сообщения, повреждённые конфиги, неожиданные выходы
  процессов).
- Глубокая защита (defence-in-depth) для чувствительных данных: редакция
  секретов *до* записи на диск, политика захвата per-app (при её реализации),
  режим файлов `0600` для пользовательских данных.
- Прозрачность вместо магии: каждое решение о заморозке прослеживаемо;
  пользователь может видеть, что сделал Froggy и почему.

Вне скоупа:

- **Злоумышленные локальные пользователи.** Любой, у кого есть shell-доступ
  к тому же Mac, может читать `~/Library/Application Support/Froggy/`,
  писать в IPC-сокет или заменить бинарник демона. Защита от этого не
  предусмотрена — та же модель доверия, что у любого пользовательского
  macOS-приложения.
- **Недоверенная сеть.** Unix socket не имеет сетевого доступа по замыслу.
  Не привязывайте его к сетевому интерфейсу; не проксируйте его через SSH
  на недоверенные хосты.
- **Скомпрометированные зависимости / supply chain.** `swift package` собирает
  из публичных реестров; атестации supply chain отсутствуют в текущей модели
  угроз. Lock-файлы закоммичены; проверяйте диффы в `Package.resolved` при
  обновлении.
- **Side channels.** Тайминг давления на память, паттерны заморозок или
  задержки генерации могут раскрыть информацию о поведении пользователя
  локальному атакующему, который может их наблюдать. Защита не предусмотрена.
- **Недоверенный ввод для MLX.** Модели загружаются с локальных путей,
  которые указывает пользователь. Содержимое моделей не валидируется —
  вредоносный файл модели теоретически может эксплуатировать MLX или среду
  выполнения инференса. Не загружайте модели из источников, которым не
  доверяете.

## Чувствительные поверхности атаки

Если вы проводите аудит или рефакторинг с учётом безопасности, начните
с этих мест:

- **`LushaBridge/Redactor.swift`** — шаг редакции перед записью контекста
  на диск. Реализован на регулярных выражениях, хрупок по замыслу (см.
  `docs/POSITIONING.md`); тщательно проверяйте изменения паттернов.
- **`VortexCore/IPC.swift`** — протокол JSON-line через Unix socket. Без
  аутентификации; опирается на права доступа файловой системы. Не добавляйте
  команды, принимающие произвольные пути к файлам, без анализа path traversal.
- **`VortexCore/Pageout.swift`** — использует `task_for_pid` и приватный
  символ `memorystatus_control` через `@_silgen_name`. Поведение зависит от
  внутренностей macOS; см. ADR 0007.
- **`Sources/FroggyMLXWorker/`** — дочерний процесс. Доверие parent-child
  предполагается; worker не изолирован в sandbox за пределами дефолта ОС.
- **`packaging/Froggy.entitlements`** — entitlements, предоставляемые при
  подписании. Не добавляйте новые entitlements без ADR.
- **`frozen.pids`** — файл в `~/Library/Application Support/Froggy/`,
  используемый для восстановления после краша. Режим `0600`. При подмене
  может заставить Froggy отправить `SIGCONT` произвольным PID при загрузке —
  но только тем, которые проходят проверки `ProcessClassifier`.

## Примечания о приватности

- Снимки экрана обрабатываются в памяти и проходят через `Redactor` перед
  любым сохранением. Сырые кадры никогда не записываются на диск.
- Скользящее контекстное окно хранит последние N редактированных снапшотов
  только в памяти.
- `freeze_stats.sqlite` (когда приземлится Mem-5) содержит bundle ID и
  временны́е метки, но не содержимое. Режим файла `0600`.
- По умолчанию ничего не отправляется за пределы устройства. Cloud-routing,
  когда будет добавлен, будет opt-in для каждого источника с отдельным
  анализом модели угроз.

## Известные ограничения

- `Redactor` реализован на регулярных выражениях и **неполон по замыслу**.
  Он перехватывает AWS-ключи, GitHub PAT, распространённые форматы токенов,
  JWT, кредитные карты с валидацией Луна. Он **не** перехватывает
  контекстно-специфические секреты (внутренние URL, имена контактов,
  медицинские данные, внутренние кодовые названия проектов). Считайте
  редакцию защитой по принципу best-effort defence-in-depth, а не гарантией.
- Entitlement `task_for_pid-allow` требуется для работы стратегии pageout
  `machVM` на сторонних процессах. Apple выдаёт его редко; стратегия `jetsam`
  по умолчанию работает без него. Смотрите `packaging/README.md` и ADR 0007.
